  `IsxMethod::EhrlichKsg`): synergy is the real `I^sx` Möbius atom, not a proxy. Discrete

  SxPID exists upstream (pid-rs 0.4.0 `sxpid.rs`) and remains an explicitly tracked wiring
  follow-up.

  `co_information = Red−Syn` exact. k/metric/tie_epsilon coherent across KSG/I^sx/geometry.
  The duplicate-free subsampling implementation is suitable as an m-sample stability probe,
  but its uncentered raw percentiles are not an n-sample confidence interval. Units are nats.
- Explicitly clean under direct verification: `manipulation.rs` Flow_gt, `physics.rs` core,
  safe-mode gating, replay verification, all three bridge transports, `pid-rerun` runlog
  surfacing, `observe.rs` wire-0.6 gate, SAFE adapter provenance, attribution probe honesty,

## Defects (prioritized)

### HIGH

1. **[FIXED 2026-07-09] Rejected RPCs poison run-log validity.**
   `crates/pid-sim/src/lib.rs` `record_rejected_rpc` wrote a `BridgeResponse` with no paired
   `BridgeRequest` (only `dispatch` recorded requests; unknown-method / malformed-JSON /
   bad-id rejection paths bypassed it), and `pid-runlog` validation hard-errors on a
   response without its request — so one `{"method":"bogus.method"}` probe made the
   canonical log `valid=false`. Empirically reproduced before the fix (`errors=2`).
   Fix: record the paired `BridgeRequest` before the rejected response; regression test
   validates the log end-to-end after a rejected RPC.

### MEDIUM (open)

2. **[FIXED 2026-07-09] ncp-observer: unbounded deques in healthy long sessions** —
   `pending_order`/`d_order` retained one stale i64 per completed seq forever.
   `enforce_bounds` now drains the stale deque front (amortized O(1)) and compacts
   outright past 2×`MAX_INFLIGHT` (strict bound even with a stuck live front); regression
   test drives MAX_INFLIGHT+500 fully-completing ticks and asserts the deques stay
   grace-window sized.
3. **[FIXED 2026-07-09] ncp-observer: duplicate `sample_id` on transport re-delivery** —
   `on_sensor`/`on_command` now carry the same `emitted_by_seq` guard the observation path
   has; re-delivered frames are dropped and counted (`redelivered_frames_dropped`).
   Regression test asserts sample-id uniqueness after re-delivery.
4. **[FIXED 2026-07-09] ncp-observer: observation frames advanced `max_seq` without
   `admit_seq`** — observations no longer move the watermark at all: D readouts are
   passengers on the control loop's clock (pairing via `d_by_seq` is unaffected), and only
   sensor/command ingress — which passes `admit_seq`'s reset discipline — advances it.
   Regression test injects `obs.seq = 1_000_000` mid-session and asserts zero resets, one
   epoch, all ticks emitted.
5. **[FIXED 2026-07-09] Bridge `request_id` collisions invalidated logs** — run-log ids are
   now unique by construction and type-unambiguous
   (`rpc_id_to_unique_request_id`: `message-{index}:{n:|s:|null}{id}`, index =
   session-monotone message counter), covering `1` vs `"1"`, id reuse by spec-valid
   clients, and multiple notifications; wire responses still echo the original id
   verbatim. The old bare renderer's doc no longer claims collisions are fine.
   Regression test fires all three collision classes and validates the log end to end.
6. **[FIXED 2026-07-09] `scene.set_object` bypasses input validation** — it deserialized
   and upserted with no `validate_object_id`/finiteness checks (contrast
   `intervention.apply`); an empty id was accepted and poisoned the next `SimSnapshot`'s
   validation. Fixed: the handler now runs the same validators (`validate_object_id`,
   `validate_pose_finite`, `validate_vec3_finite`) before `upsert_object`; regression test
   covers the empty-id rejection and asserts the log stays valid end to end. Note: the
   finiteness checks are defense-in-depth for programmatic callers — JSON itself cannot
   carry NaN/Inf (serde_json rejects them at parse), so the reachable-via-RPC case was the
   empty id.
7. **[ADDRESSED 2026-07-09] Permutation nulls assume exchangeable rows** — the harness used
   `permutation_rows_pvalue` (full-row Fisher–Yates) for per-step captures where
   within-episode autocorrelation violates exchangeability → anti-conservative p-values.
   Fixed: the pid-rs submodule is advanced to 8a5a9dd and the harness exposes
   `--permutation-scheme full-shuffle|circular-shift` (config field
   `permutation_scheme`); `circular-shift` uses `PermutationScheme::CircularShift` with
   `min_shift = --uncertainty-block-size` (the dependence length already sized for the
   bootstrap), and the chosen null is recorded in the uncertainty artifact
   (`"permutation_scheme"`). Default remains `full-shuffle` (bit-identical to prior
   releases); **use `circular-shift` for real per-step captures.** Remaining refinement:
   an episode-block permutation (permute source across episodes, preserving within-episode
   structure) once real episode boundaries exist in captures.
8. **[ADDRESSED 2026-07-09] `discrete-pls` mode omitted its preregistered mitigations**
   (grandplan §6.2 leakage/fitted-preprocessing). Now implemented: (a) **per-source LOO-CV Q² component
   selection** (`--pls-components cv[:MAX]`, via `pls_cv_select_components`; chosen counts
   and Q² recorded in the summary's `pls_selection`; fixed counts remain available and
   default); (b) **the shuffled-target permutation control** — every discrete-pls screen
   carries `pls_shuffled_target_control`, the identical pipeline (PLS fit + discrete PID)
   run against a seeded row-shuffle of `A` (seed recorded): with true dependence destroyed,
   its atoms are the selection-inflation floor, and the docs instruct reading the real
   screen relative to it, screening-only. The test suite demonstrated the point on the
   small fixture: in the saturated-binning regime the control **equals** the real screen —
   the numbers there are pure artifact, exactly what the control exists to expose (the
   per-pair `discrete_saturation` diagnostic flags the same regime). Remaining (tracked):
   held-out-transformed PID estimates — meaningful once the H1 per-episode endpoint
   machinery (item 9's open half) exists.
9. **[UPDATED 2026-07-10 — still blocking] README/EXPERIMENTS overstate analysis readiness**
   — there is still no leakage-safe episode-local H1 feature path. A §6.8 (power/design) simulator now
   exists, but audit found effect-calibration, H4-direction, ICC/nesting, measurement-error,
   and selected-design type-I defects; its published counts are withdrawn as capture
   requirements. The endpoint and nested capture-design machinery remain to build.
10. **[ADDRESSED 2026-07-09] Subsample CIs inherit m-sample estimator bias** — percentile intervals from m=n/2
    subsamples are mis-centered (KSG/I^sx bias grows as samples shrink), not just wider;
    CIs can exclude the point estimate. Fixed: `OfflineVldaAtomCi` now exposes `boot_mean`
    and the precomputed `bias_vs_point` gap with a doc-comment giving the reading (a gap
    large relative to the CI width flags m-dependent bias; treat intervals as
    width-conservative variability bands, not calibrated coverage).
