## Defects (prioritized)

### HIGH

1. **[FIXED 2026-07-09] Rejected RPCs poison run-log validity.**
   `crates/pid-sim/src/lib.rs` `record_rejected_rpc` wrote a `BridgeResponse` with no paired
   `BridgeRequest` (only `dispatch` recorded requests; unknown-method / malformed-JSON /
   bad-id rejection paths bypassed it), and `pid-runlog` validation hard-errors on a
   response without its request — so one `{"method":"bogus.method"}` probe made the
   canonical log `valid=false`. Empirically reproduced before the fix (`errors=2`).
   Fix: record the paired `BridgeRequest` before the rejected response; regression test
   validates the log end-to-end after a rejected RPC.

### MEDIUM (open)

2. **ncp-observer: unbounded deques in healthy long sessions** — `pending_order`/`d_order`
   (`crates/ncp-observer/src/lib.rs:393/415/464`) are drained only by `enforce_bounds` when
   maps exceed `MAX_INFLIGHT`; normal removal paths never pop them → one stale i64 per seq
   forever in steady state. Fix: opportunistic front-pop on removal or cap deque length.
3. **ncp-observer: duplicate `sample_id` on transport re-delivery** — `on_sensor`/
   `on_command` (lib.rs:392-395/413-416) lack the `emitted_by_seq` guard the observation
   path has (:443); a re-delivered pair re-emits an identical `sample_id`, double-counting
   a (V,L,D,A) row. Fix: guard both handlers.
4. **ncp-observer: observation frames advance `max_seq` without `admit_seq`** (lib.rs:467-469)
   — one inflated `obs.seq` misclassifies later frames as session resets and prematurely
   emits `recency_fallback` samples. Fix: same reset-margin discipline as other ingress.
5. **Bridge `request_id` collisions invalidate logs** — `crates/pid-bridge/src/lib.rs:210-220`
   doc says colliding renderings are fine; the validator errors on duplicates
   (pid-runlog:1008/1028). Fix: type-disambiguate ids or reconcile doc/validator.
6. **[FIXED 2026-07-09] `scene.set_object` bypasses input validation** — it deserialized
   and upserted with no `validate_object_id`/finiteness checks (contrast
   `intervention.apply`); an empty id was accepted and poisoned the next `SimSnapshot`'s
   validation. Fixed: the handler now runs the same validators (`validate_object_id`,
   `validate_pose_finite`, `validate_vec3_finite`) before `upsert_object`; regression test
   covers the empty-id rejection and asserts the log stays valid end to end. Note: the
   finiteness checks are defense-in-depth for programmatic callers — JSON itself cannot
   carry NaN/Inf (serde_json rejects them at parse), so the reachable-via-RPC case was the
   empty id.
7. **[ADDRESSED 2026-07-09] Permutation nulls assume exchangeable rows** — the harness used
   `permutation_rows_pvalue` (full-row Fisher–Yates) for per-step captures where
   within-episode autocorrelation violates exchangeability → anti-conservative p-values.
   Fixed: the pid-rs submodule is advanced to 8a5a9dd and the harness exposes
   `--permutation-scheme full-shuffle|circular-shift` (config field
   `permutation_scheme`); `circular-shift` uses `PermutationScheme::CircularShift` with
   `min_shift = --uncertainty-block-size` (the dependence length already sized for the
   bootstrap), and the chosen null is recorded in the uncertainty artifact
   (`"permutation_scheme"`). Default remains `full-shuffle` (bit-identical to prior
   releases); **use `circular-shift` for real per-step captures.** Remaining refinement:
   an episode-block permutation (permute source across episodes, preserving within-episode
   structure) once real episode boundaries exist in captures.
8. **`discrete-pls` mode omits its own preregistered mitigations** (grandplan §8.2.3 step 5):
   in-sample PLS fits per screen, no shuffled-target control, no held-out-transformed PID,
   fixed `--pls-components` despite `pls_cv_select_components` existing upstream → numbers
   optimistically biased by construction. Fix: cross-fitting + shuffled-target control + CV
   component selection, or gate outputs as screening-only.
9. **[ADDRESSED 2026-07-09 — wording qualified; endpoint machinery still to build] README/EXPERIMENTS overstate analysis readiness** — "wired end to end" / "analysis
   tooling is ready": no per-episode/windowed PID feature extraction, no H1
   primary-endpoint code (held-out episode-level ΔAUROC with paired block bootstrap,
   grandplan §14.8.1), no §14.8.3 power-analysis script. Fix wording or implement.
10. **[ADDRESSED 2026-07-09] Subsample CIs inherit m-sample estimator bias** — percentile intervals from m=n/2
    subsamples are mis-centered (KSG/I^sx bias grows as samples shrink), not just wider;
    CIs can exclude the point estimate. Fixed: `OfflineVldaAtomCi` now exposes `boot_mean`
    and the precomputed `bias_vs_point` gap with a doc-comment giving the reading (a gap
    large relative to the CI width flags m-dependent bias; treat intervals as
    width-conservative variability bands, not calibrated coverage).
